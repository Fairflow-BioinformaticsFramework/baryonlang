# run_function.py

`run_function.py` is a lightweight helper script designed to test and execute self-contained Python functions generated by **Baryon** directly from the command line.

When Baryon compiles a `.bala` file into a Python target using the `--generate_function` (`-f`) flag, it wraps the entire tool execution logic inside a single, reusable Python function. This script automates the process of loading that function dynamically and passing command-line arguments to it.

---

## How It Works

The script leverages Python's internal reflection capabilities (`importlib`) to perform the following operations:
1. **Dynamic Importing:** It extracts the module name from the provided script path and dynamically loads it into the runtime environment.
2. **Function Resolution:** It searches for a function inside the loaded file that matches the filename exactly (e.g., inside `my_analysis.py`, it looks for a function named `my_analysis`).
3. **Argument Forwarding:** It captures all subsequent command-line arguments as a list of strings and unpacks them directly into the target function using the `*args` splat operator.
4. **Exit Code Propagation:** If the function returns an integer or an explicit exit status, it propagates that value back to the system shell.

---

## Prerequisites

- Python 3.x
- UTF-8 compatible terminal environment (the script automatically reconfigures standard output to use `utf-8` encoding).

---

## Usage

Run the script from your terminal by passing the path to the generated Python tool script followed by any positional arguments required by that specific tool.

```bash
python run_function.py <generated_function.py> [arg1 arg2 arg3 ...]
```

# run_function.R

`run_function.R` is a lightweight helper script designed to test and execute self-contained R functions generated by **Baryon** directly from the command line.

When Baryon compiles a `.bala` file into an R target using the `--generate_function` (`-f`) flag, it wraps the entire tool execution logic inside a single, reusable R function. This script automates the process of loading that function dynamically and passing command-line arguments to it.

---

## How It Works

The script leverages R's internal dynamic execution capabilities to perform the following operations:
1. **Dynamic Sourcing:** It loads the target R script into the current environment using the `source()` function.
2. **Function Resolution:** It determines the target function name based on the base name of the script (e.g., inside `my_analysis.R`, it looks for an R function named `my_analysis`).
3. **Argument Forwarding:** It captures all subsequent command-line arguments using `commandArgs(trailingOnly = TRUE)` and forwards them seamlessly as independent positional strings to the target function using `do.call`.
4. **Exit Code Propagation:** It intercepts the underlying return status of the executed function and bubbles it back up to the system shell.

---

## Prerequisites

- R environment (accessible via the `Rscript` executable on your system PATH).
- A terminal environment capable of handling ANSI color escape sequences for logging.

---

## Usage

Run the script from your terminal using `Rscript`, passing the path to the generated R tool script followed by any positional arguments required by that specific tool.

```bash
Rscript run_function.R <path_to_generated_script.R> [arg1 arg2 arg3 ...]
```

# run_function.sh

`run_function.sh` is a lightweight helper script designed to test and execute self-contained Bash functions generated by **Baryon** directly from the command line.

When Baryon compiles a `.bala` file into a Bash target using the `--generate_function` (`-f`) flag, it wraps the entire tool execution logic inside a single, reusable Bash function. This script automates the process of loading that function dynamically and passing command-line arguments to it.

---

## How It Works

The script leverages Bash's native runtime command execution and environmental scaling to perform the following operations:
1. **Dynamic Sourcing:** It imports the target Bash script into the current shell execution environment using the `source` built-in command.
2. **Function Resolution:** It determines the target function name based on the base name of the file (e.g., inside `my_analysis.sh`, it looks for a Bash function named `my_analysis`).
3. **Argument Forwarding:** It shifts out the script path from the positional parameters and forwards all subsequent command-line arguments seamlessly using the `"$@"` array descriptor. This ensures that spaces, quotes, and complex strings are preserved intact.
4. **Exit Code Propagation:** It captures the exact return code (`$?`) returned by the execution of the internal function and bubbles it back up to the system shell interface.

---

## Prerequisites

- A POSIX-compliant terminal environment (such as Linux Bash, macOS Terminal, or WSL on Windows).
- Execution permissions set on the runner script (`chmod +x run_function.sh`).

---

## Usage

Run the script from your terminal by passing the path to the generated Bash tool script followed by any positional arguments required by that specific tool.

```bash
./run_function.sh <path_to_generated_script.sh> [arg1 arg2 arg3 ...]
```
